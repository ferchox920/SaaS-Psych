# S0_1 - Bootstrap Base (Go + Echo)

## Paso
`S0_1`

## Objetivo
Crear una base tecnica minima, compilable y testeable para SessionFlow: estructura de `apps/api`, servidor HTTP con healthcheck, configuracion por entorno, conexion inicial a Postgres y migraciones base multi-tenant.

## Estado previo detectado
- Solo existia `PROJECT_GUIDELINES.md` como fuente de verdad.
- No existia `apps/api` ni modulo Go.
- No habia migraciones, ni `docker-compose.yml`.
- No existia `PROGRESS/PROGRESS_INDEX.md` ni historial de pasos.

## Cambios realizados
- Se inicializo `apps/api` con Go 1.22 y dependencias Echo + pgx.
- Se implemento bootstrap de servidor (`cmd/server/main.go`) con apagado graceful por senales.
- Se agrego servidor Echo con endpoint `GET /health` y middleware de recover.
- Se agrego carga de configuracion por entorno (`APP_ENV`, `HTTP_PORT`, `DATABASE_URL`).
- Se agrego inicializacion de pool Postgres con `pgxpool` y `Ping`.
- Se crearon migraciones iniciales para `tenants`, `users`, `user_roles`, `refresh_tokens`.
- Se agregaron tests minimos para handler de health y config.
- Se agregaron `.env.example`, `docker-compose.yml` y el indice de progreso.

## Decisiones y trade-offs
- Se eligio Echo por alineacion con la recomendacion del proyecto y simplicidad de middleware.
- Se permitio iniciar sin DB si `DATABASE_URL` esta vacio para facilitar boot local incremental.
- Se limito el alcance a bootstrap, sin auth/RBAC/usecases de negocio para mantener entrega pequena.
- Se incluyeron carpetas `domain` y `usecase` con `.gitkeep` para marcar arquitectura sin sobre-implementar.
