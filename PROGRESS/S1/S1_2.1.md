# S1_2.1 - Normalizacion de Auth JSON a snake_case

## Paso
`S1_2.1`

## Objetivo
Normalizar el contrato JSON de respuestas de auth (`login` y `refresh`) a `snake_case` para alinear con la convencion del proyecto y evitar mezcla de estilos.

## Estado previo detectado
- El ultimo paso completado era `S1_2`.
- El flujo auth (`login -> me -> refresh -> logout`) ya estaba cubierto por test de integracion HTTP.
- Las respuestas `login/refresh` salian en `PascalCase` por serializacion default de structs Go.
- Otros contratos de error y `me` ya estaban en `snake_case`.

## Cambios realizados
- Se agregaron tags JSON en `auth.LoginOutput`:
  - `access_token`
  - `refresh_token`
  - `token_type`
  - `expires_in`
- Se actualizo test de integracion para parsear `snake_case` en respuestas de `login` y `refresh`.
- Se validaron pruebas y build para asegurar compatibilidad.

## Decisiones y trade-offs
- Se eligio cambio de contrato ahora para homogeneizar API antes de sumar endpoints de negocio.
- Se toco solo serializacion de salida; no se altero logica de negocio ni persistencia.
- Potencial impacto: clientes que consumian `PascalCase` deben ajustarse (aceptable en etapa temprana).
