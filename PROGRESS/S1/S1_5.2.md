# S1_5.2 - Helper compartido para mapeo de errores de dominio en handlers

## Paso
`S1_5.2`

## Objetivo
Centralizar el mapeo de errores de dominio en un helper reutilizable para reducir duplicacion entre handlers (`auth` y `audit`) manteniendo respuestas HTTP consistentes.

## Estado previo detectado
- Ultimo paso completado: `S1_5.1`.
- `auth` y `audit` ya usaban `errors.Is(...)`, pero cada handler definia su propio mapeo y fallback.
- El contrato de errores tipados ya existia en `internal/domain/errors`.
- No habia helper comun para mapear `ErrValidation`, `ErrUnauthorized`, `ErrForbidden` y fallback `500`.
- No habia tests unitarios dedicados al mapeo compartido.

## Cambios realizados
- Se agrego `error_mapper.go` en `internal/http/handlers` con:
  - `handleDomainError(...)`
  - `domainErrorMapping`
  - `defaultAuthErrorMappings()`
  - `defaultAuditErrorMappings()`
- Se simplifico `AuthHandler.handleAuthError` para delegar el mapeo al helper compartido.
- Se actualizo `AuditHandler.List` para delegar su manejo de errores del servicio al helper compartido.
- Se agregaron tests unitarios de mapper:
  - conserva mensaje explicito para `ErrValidation`
  - fallback a `500 internal_error` para errores no mapeados.

## Decisiones y trade-offs
- Se mantuvieron mapeos por dominio de handler (`auth` y `audit`) para conservar flexibilidad de mensajes/codigos sin duplicar logica base.
- El fallback interno ahora es uniforme y no expone detalles de errores internos en respuesta HTTP.
- Se asumio `S1_5.2` como paso actual por no venir completada la seccion `TAREA ACTUAL` en el prompt.
