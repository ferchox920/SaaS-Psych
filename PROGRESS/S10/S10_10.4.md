# S10_10.4 - Alerting Prometheus versionado (local)

## Paso
`S10_10.4`

## Objetivo
Agregar reglas de alerting Prometheus versionadas para entorno local (error rate, latencia p95, API down) y documentar su verificacion.

## Estado previo detectado
- Prometheus y Grafana ya estaban en stack local.
- No existian archivos de reglas de alertas versionados.
- `prometheus.yml` no referenciaba `rule_files`.

## Cambios realizados
- `deploy/observability/prometheus-rules.yml` (nuevo):
  - `SessionFlowApiDown`: `up{job="sessionflow-api"} == 0` por `1m`.
  - `SessionFlowHigh5xxErrorRate`: ratio 5xx > `5%` por `5m`.
  - `SessionFlowHighP95Latency`: p95 global > `750ms` por `10m`.
- `deploy/observability/prometheus.yml`:
  - agregado `rule_files: ["/etc/prometheus/rules.yml"]`.
- `docker-compose.yml`:
  - montaje de reglas en servicio `prometheus`:
    - `./deploy/observability/prometheus-rules.yml:/etc/prometheus/rules.yml:ro`
- `README.md`:
  - se documenta archivo de reglas.
  - se agregan rutas de verificacion (`/rules`, `/alerts`).
  - se agregan thresholds y guia minima para probar `ApiDown` localmente.
  - se explicita que no se incluye Alertmanager en este stack minimo (estructura preparada).
