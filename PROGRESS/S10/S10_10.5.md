# S10_10.5 - Runbook operativo minimo para incident response

## Paso
`S10_10.5`

## Objetivo
Crear runbook operativo minimo en `docs/RUNBOOK.md` cubriendo healthchecks, logs correlacionados, metricas, trazas y fallas comunes (DB/Redis/OTEL collector), con comandos `make` y `docker compose`.

## Estado previo detectado
- README contenia comandos operativos y observabilidad, pero no habia runbook de incident response consolidado.
- Faltaba una guia secuencial para diagnostico y recuperacion rapida en incidentes comunes.

## Cambios realizados
- Nuevo documento `docs/RUNBOOK.md` con enfoque operativo:
  1. Healthchecks (`/health`, `/metrics`, targets/alerts en Prometheus).
  2. Logs correlacionados (`request_id`, `trace_id`, `span_id`, `tenant_id`).
  3. Metricas (queries PromQL para requests, 5xx, p95 y alertas versionadas).
  4. Trazas en Jaeger (flujo de investigacion por `trace_id`).
  5. Fallas comunes y acciones:
     - Postgres down
     - Redis down
     - OTel collector down
  6. Checklist de cierre de incidente.
  7. Comandos rapidos (`docker compose`, `make test`, `make test-integration-db`).
- No se modifico codigo ni comportamiento de runtime.
