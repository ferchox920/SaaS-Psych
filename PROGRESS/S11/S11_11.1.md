# S11_11.1 - Contrato de errores HTTP unificado en middlewares

## Paso
`S11_11.1`

## Objetivo
Unificar el contrato de errores HTTP en middlewares de `tenant`, `auth` y `rate_limit` usando el helper central (`http/response.WriteError`), manteniendo status codes actuales y agregando soporte de `error.details` opcional con metadata (`field`, `reason`).

## Estado previo detectado
- Los handlers ya usaban el helper central de errores (`writeAPIError` -> `response.WriteError`).
- Los middlewares usaban un helper local `writeError` en `tenant.go` con payload parcial (`code`, `message`) y sin `details`.
- `auth.go` y `rate_limit.go` dependian de ese helper local, por lo que el contrato de error no era consistente.

## Cambios realizados
- `apps/api/internal/http/middleware/tenant.go`:
  - Reemplazo de helper local por `httpresponse.WriteError`.
  - Se removio el helper local `writeError`.
  - Se agregaron `details` en errores de validacion de tenant:
    - missing header -> `{field: "X-Tenant-ID", reason: "required"}`
    - invalid format -> `{field: "X-Tenant-ID", reason: "invalid_uuid"}`
- `apps/api/internal/http/middleware/auth.go`:
  - Reemplazo de respuestas de error por `httpresponse.WriteError`.
  - Se agregaron `details` para errores de header/token:
    - missing authorization -> `{field: "Authorization", reason: "required"}`
    - invalid bearer format -> `{field: "Authorization", reason: "invalid_bearer_format"}`
    - invalid/expired token -> `{field: "Authorization", reason: "invalid_or_expired_token"}`
  - Se mantuvieron status codes y mensajes existentes.
- `apps/api/internal/http/middleware/rate_limit.go`:
  - Reemplazo de respuestas de error por `httpresponse.WriteError`.
  - Se agrego `details` en `429`:
    - `{field: "ip", reason: "limit_exceeded"}`
- Tests minimos agregados/extendidos:
  - `tenant_test.go`: valida envelope y `details` en missing `X-Tenant-ID`.
  - `auth_test.go`: nuevo test para token invalido que valida envelope y `details`.
  - `rate_limit_test.go`: valida envelope y `details` en rate limit exceeded.
- OpenAPI:
  - `POST /api/v1/auth/login` ahora explicita `429` (`TooManyRequestsError`).
  - Se agrego ejemplo de `UnauthorizedError` con `details` para token invalido/expirado.
  - Nuevo componente `TooManyRequestsError` con ejemplo de `details`.

## Decisiones y trade-offs
- Cambio acotado a middlewares para reducir riesgo y evitar refactor masivo.
- Se reutilizo `response.WriteError` (fuente unica del envelope) sin introducir nuevas capas.
- Se agregaron `details` solo en errores donde aportan diagnostico inmediato.
