# S11_11.2 - Alertmanager local integrado al stack de alertas

## Paso
`S11_11.2`

## Objetivo
Completar el ciclo de alertas local agregando Alertmanager al stack reproducible, conectando Prometheus para envio de alertas y documentando operacion/prueba en README y RUNBOOK.

## Estado previo detectado
- Existian reglas de alertas en Prometheus (`deploy/observability/prometheus-rules.yml`) y visualizacion en `/alerts`.
- No habia servicio Alertmanager en `docker-compose.yml`.
- Prometheus no tenia seccion `alerting.alertmanagers` para enrutar alertas.
- README y RUNBOOK explicitaban que no se incluia Alertmanager.

## Cambios realizados
- `docker-compose.yml`:
  - Se agrego servicio `alertmanager` (`prom/alertmanager:v0.27.0`) con config montada y puerto `9093`.
  - Se agrego volumen persistente `alertmanager_data`.
  - `prometheus` ahora declara dependencia hacia `alertmanager`.
- `deploy/observability/alertmanager.yml` (nuevo):
  - Config minima versionada con route por defecto y receiver `dummy-webhook`.
  - Receiver webhook placeholder para entorno local/dev reproducible (`host.docker.internal:65535`).
- `deploy/observability/prometheus.yml`:
  - Se agrego `alerting.alertmanagers` apuntando a `alertmanager:9093`.
- `README.md`:
  - Se documento `alertmanager` en servicios locales y comandos de levantado.
  - Se agrego referencia al archivo `deploy/observability/alertmanager.yml`.
  - Se incluyo verificacion de Alertmanager (`http://localhost:9093`) y prueba local de alerta `SessionFlowApiDown` con recepcion en Alertmanager.
- `docs/RUNBOOK.md`:
  - Se actualizo stack de observabilidad para incluir Alertmanager.
  - Se agrego check de UI `:9093`.
  - Se documento pipeline Prometheus -> Alertmanager y prueba end-to-end de alerta.
  - Se agrego falla comun `Alertmanager down` con acciones de diagnostico/recuperacion.

## Decisiones y trade-offs
- Se eligio receiver webhook placeholder para mantener setup local simple y sin dependencias externas.
- El webhook dummy puede fallar entrega (esperado), pero valida pipeline real de enrutamiento Prometheus -> Alertmanager en entorno local.
- No se agrego Alertmanager production-grade (routing avanzado, inhibiciones, notifiers reales) por alcance del paso.
