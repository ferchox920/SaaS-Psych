# S11_11.4 - Bootstrap local reproducible para herramienta migrate

## Paso
`S11_11.4`

## Objetivo
Mejorar la experiencia local agregando un flujo reproducible para instalar/verificar `migrate` via `make tools`, incorporar `make db-up` al flujo recomendado y documentar compatibilidad Linux/macOS con nota para Windows.

## Estado previo detectado
- El `Makefile` no tenia target de bootstrap de herramientas (`tools`).
- `migrate` se asumia preinstalado manualmente.
- No habia target simple para levantar DB local (`db-up`) desde Make.
- README no mostraba flujo recomendado unificado `make tools -> make db-up -> make migrate-up -> make test`.

## Cambios realizados
- `Makefile`:
  - Nuevo `MIGRATE_VERSION ?= v4.18.3` alineado con version usada en CI.
  - Nuevo target `tools` que instala/verifica `migrate` usando script reproducible.
  - Nuevo target `db-up` (`docker compose up -d postgres redis`).
  - Nuevo target `db-down` (`docker compose stop postgres redis`) para operacion local.
- Nuevo script `scripts/install_migrate.sh`:
  - Soporta Linux/macOS (deteccion de OS/arch).
  - Descarga release oficial de `golang-migrate` segun plataforma.
  - Instala en `/usr/local/bin` (o `MIGRATE_INSTALL_DIR`) usando `sudo` solo si hace falta.
  - Si `migrate` ya existe, solo verifica version y no reinstala.
- `README.md`:
  - Se actualizo requisitos para indicar que `migrate` se instala/verifica con `make tools`.
  - Se agrego flujo recomendado de bootstrap:
    - `make tools`
    - `make db-up`
    - `make migrate-up`
    - `make test`
  - Se agrego compatibilidad de `make tools` (Linux/macOS) y nota Windows con instalacion manual (`choco`/`scoop`).
