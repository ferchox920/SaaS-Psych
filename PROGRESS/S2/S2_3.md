# S2_3 - Session Notes (privacidad + RBAC)

## Paso
`S2_3`

## Objetivo
Implementar `P2` de `Session Notes` con regla de privacidad: notas `is_private=true` visibles solo para autor y roles `owner/admin`, con endpoints y tests.

## Estado previo detectado
- `S2_2` completo con appointments y no-solapamiento.
- Auth + tenancy + RBAC operativo en middleware.
- No existia tabla `session_notes` ni casos de uso/endpoints de notas.

## Cambios realizados
- Se agrego migracion `session_notes` con `tenant_id`, `appointment_id`, `author_user_id`, `body`, `is_private`, timestamps e indices.
- Se agrego dominio `sessionnote` con:
  - validaciones de entidad
  - regla `CanView(note, requesterUserID, requesterRole)`.
- Se implemento `usecase/sessionnote` con:
  - `Create`
  - `Get` (aplica visibilidad privada)
  - `ListByAppointment` (filtra por visibilidad para requester)
  - chequeo de existencia de appointment por tenant antes de crear.
- Se implemento repositorio Postgres `SessionNoteRepository` con filtro por `tenant_id` en todas las queries.
- Se agrego `SessionNoteHandler` y rutas:
  - `POST /api/v1/appointments/:appointment_id/notes`
  - `GET /api/v1/appointments/:appointment_id/notes`
  - `GET /api/v1/notes/:id`
- Se usa `PrincipalFromContext` para aplicar privacidad por `user_id` y `role`.
- Se cableo en `server` y `main`.
