# S5_5.2 - Pipeline CI con lint, test/build e integration DB

## Paso
`S5_5.2`

## Objetivo
Crear pipeline de CI en GitHub Actions que ejecute:
- `go test ./...`
- `golangci-lint`
- `go build`
- job de integration tests con Postgres + Redis activando tests opt-in por env.

## Estado previo detectado
- No existia `.github/workflows/ci.yml`.
- Tests locales estaban estables con `go test ./...`.
- Integration tests Postgres estaban en modo opt-in (`RUN_PG_INTEGRATION=1`).
- No habia configuracion de `golangci-lint` en el repo.
- Makefile ya definia flujo de migraciones y comando de integration local.

## Cambios realizados
- Se agrego workflow `.github/workflows/ci.yml` con 3 jobs:
  - `test-build`: `go test ./...` y `go build ./cmd/server`.
  - `lint`: `golangci-lint` con action oficial.
  - `integration-db`: levanta servicios `postgres` y `redis`, aplica migraciones y ejecuta tests de integracion.
- Se agrego configuracion minima de linter en `apps/api/.golangci.yml`:
  - linters: `govet`, `ineffassign`, `errcheck`.
- En job de integration:
  - variables: `DATABASE_URL`, `REDIS_URL`, `RUN_PG_INTEGRATION=1`.
  - instalacion de `migrate` CLI en runner.
  - `migrate up` sobre `apps/api/migrations`.
  - corrida de tests `./internal/http` y `./internal/infra/db`.
- Se documento en README el pipeline y las variables requeridas por CI.

## Archivos tocados
- `.github/workflows/ci.yml`
- `apps/api/.golangci.yml`
- `README.md`
- `PROGRESS/PROGRESS_INDEX.md`

## Migraciones

- En CI se ejecutan migraciones existentes antes de integration tests.
