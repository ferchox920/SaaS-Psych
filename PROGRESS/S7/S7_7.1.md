# S7_7.1 - Tracing HTTP con OpenTelemetry (OTLP por env)

## Paso
`S7_7.1`

## Objetivo
Implementar tracing HTTP unicamente con OpenTelemetry:
- middleware Echo con un span por request
- propagacion de contexto
- exporter OTLP configurable por env
- documentacion local con collector simple
- tests minimos de provider/middleware

## Estado previo detectado
- Observabilidad existente: request logging y metricas Prometheus (`/metrics`) ya activas.
- No existia tracing OpenTelemetry en servidor ni middleware HTTP.
- `config` no exponia variables OTEL.
- `README` no explicaba ejecucion local con collector para trazas.

## Cambios realizados
- Se extendio `config.Config` con:
  - `OTEL_SERVICE_NAME`
  - `OTEL_EXPORTER_OTLP_ENDPOINT`
  - `OTEL_TRACES_EXPORTER`
  - `OTEL_RESOURCE_ATTRIBUTES`
- Se agrego `observability.NewTracerProvider(...)`:
  - crea `TracerProvider` con recurso (`service.name` + atributos custom)
  - soporta exporter `none` (default) y `otlp`
  - configura propagador global (`TraceContext` + `Baggage`)
  - valida endpoint cuando exporter = `otlp`
- Se agrego middleware `RequestTracing(...)` en Echo:
  - extrae contexto de headers entrantes
  - crea span por request (`METHOD route`)
  - setea atributos HTTP basicos y status
  - propaga contexto a la request para siguientes capas
- Se integro el middleware de tracing en `server.go` sin romper logging ni metrics.
- Se cableo en `main.go`:
  - inicializacion de tracer provider
  - middleware HTTP de tracing
  - shutdown del tracer provider en cierre del server
- Se agregaron tests minimos:
  - provider se crea y apaga (sin exporter)
  - middleware genera span por request
- Se actualizo `.env.example` y `README.md` con variables OTEL y ejemplo de collector local.
- Se agrego `docs/otel-collector-config.yaml` para collector minimo.

## Comandos para correr
```bash
cd apps/api
go test ./...
go run ./cmd/server
```

Activar OTLP local:
```bash
OTEL_TRACES_EXPORTER=otlp
OTEL_EXPORTER_OTLP_ENDPOINT=localhost:4317
OTEL_SERVICE_NAME=sessionflow-api
OTEL_RESOURCE_ATTRIBUTES=deployment.environment=local,service.version=dev
```

Collector simple:
```bash
docker run --rm -p 4317:4317 -p 4318:4318 \
  -v ${PWD}/docs/otel-collector-config.yaml:/etc/otelcol-contrib/config.yaml \
  otel/opentelemetry-collector-contrib:0.104.0
```

## Decisiones y trade-offs
- Se dejo `OTEL_TRACES_EXPORTER=none` por defecto para no romper entornos sin collector.
- Se limito el alcance a HTTP para mantener cambio pequeno y estable.
- Se uso collector simple con exporter `debug` para validacion local rapida.
