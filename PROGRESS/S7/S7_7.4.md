# S7_7.4 - Stack local de tracing (OTel Collector + Jaeger) en Docker Compose

## Paso
`S7_7.4`

## Objetivo
Agregar stack local de tracing en `docker-compose` con:
- OpenTelemetry Collector
- backend de trazas (Jaeger)
- config del collector en `deploy/observability`
- documentacion operativa en README

## Estado previo detectado
- Tracing HTTP/DB/Redis ya instrumentado en aplicacion (S7.1-S7.3).
- No existia stack de tracing persistente en `docker-compose.yml`.
- Habia config de collector en `docs/`, pero no integrada al compose principal.

## Cambios realizados
- Se extendio `docker-compose.yml` sin romper `postgres` ni `redis`:
  - servicio `otel-collector`
  - servicio `jaeger`
- Se agrego config del collector en:
  - `deploy/observability/otel-collector.yaml`
- Pipeline del collector:
  - receiver `otlp` (grpc/http)
  - processor `batch`
  - exporter `otlp` hacia `jaeger:4317`
  - exporter `debug` para inspeccion local
- Se actualizo README con pasos completos:
  1. levantar stack (`docker compose up -d otel-collector jaeger`)
  2. setear env OTEL (`OTEL_EXPORTER_OTLP_ENDPOINT=localhost:4317`)
  3. generar requests
  4. abrir Jaeger (`http://localhost:16686`) y buscar `sessionflow-api`

## Decisiones y trade-offs
- Se eligio Jaeger all-in-one por simplicidad local (UI + storage + ingest en un servicio).
- Se mantuvo stack liviano (sin Tempo/Grafana en este paso) para evitar complejidad operativa.
- Se preservo compatibilidad del compose actual, agregando servicios sin alterar los existentes.
