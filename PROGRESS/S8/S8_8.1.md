## Cambios realizados
- Se agrego subseccion `Correlacion logs-traces` dentro de la seccion de tracing.
- Se documento flujo operativo:
  1. localizar request por `request_id` en logs
  2. extraer `trace_id`
  3. ubicar traza en Jaeger (`sessionflow-api`)
  4. analizar spans HTTP/DB/Redis del request
- Se agrego ejemplo realista de log JSON con `trace_id/span_id`.
- Se agrego mini guion de demo para reproducir una traza completa con `curl`.
- Se mantuvo alcance solo documental (sin cambios de codigo).
