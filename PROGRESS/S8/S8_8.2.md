# S8_8.2 - Especificacion OpenAPI 3.0 del API actual

## Paso
`S8_8.2`

## Objetivo
Crear especificacion OpenAPI 3.0 para el API actual (auth, clients, appointments, session notes, audit), incluyendo:
- esquemas request/response
- autenticacion Bearer JWT
- header `X-Tenant-ID`
- ejemplos basicos de payload

## Estado previo detectado
- No existia archivo OpenAPI en `docs/`.
- Endpoints ya estaban consolidados en `internal/http/server.go` y handlers.
- README no referenciaba una especificacion formal del contrato HTTP.

## Cambios realizados
- Se creo `docs/openapi.yaml` (OpenAPI 3.0.3) con:
  - paths reales implementados:
    - `/health`, `/metrics`
    - `/api/v1/auth/*`
    - `/api/v1/clients*`
    - `/api/v1/appointments*`
    - `/api/v1/appointments/{appointment_id}/notes`
    - `/api/v1/notes/{id}`
    - `/api/v1/audit`
  - `securitySchemes`:
    - `bearerAuth` (JWT)
    - `tenantHeader` (`X-Tenant-ID`)
  - schemas de request/response y errores estandar
  - ejemplos basicos de payload para login, client, appointment, session note
  - query params de auditoria y rango de appointments (`from/to` RFC3339)
- Se actualizo README para linkear la especificacion:
  - `docs/openapi.yaml`
